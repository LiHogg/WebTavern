# WebTavern

WebTavern — дипломный веб-проект для поиска заведений, просмотра схемы зала и бронирования столов. Система объединяет клиентский каталог, кабинет владельца, менеджерский контур, уведомления, отзывы, карты, оформление заведений и демонстрационный платёжный сценарий.

## Важно про `.env`

Файл `.env` в этой сборке уже настроен для локальной демонстрации проекта. Его не нужно пересоздавать.

Не выполнять поверх рабочей конфигурации:

```bash
cp .env.example .env
```

Это может сбросить реальные настройки почты, SMS и карт.

Текущая рабочая конфигурация:

- сайт открывается через `http://localhost:8080`;
- SMTP-уведомления отправляются через Mail.ru;
- SMS-провайдер — SMS.RU (`SMS_PROVIDER=smsru`);
- карты — Yandex Maps с fallback на локальную карту;
- база данных — PostgreSQL в Docker volume;
- Redis используется для фоновых задач и WebSocket-слоя.

## Быстрый запуск

Из корня проекта:

```bash
docker compose --env-file .env -f infra/compose/docker-compose.local.yml -p webtavern-platform up --build
```

Открывать:

- сайт: `http://localhost:8080`;
- backend/API: `http://localhost:8000`;
- Django admin: `http://localhost:8000/admin/`;
- MailHog может подниматься в compose для совместимости, но рабочая почта настроена через SMTP Mail.ru.

Остановка без удаления данных:

```bash
docker compose --env-file .env -f infra/compose/docker-compose.local.yml -p webtavern-platform down
```

Полный сброс базы и повторное заполнение демо-данными:

```bash
docker compose --env-file .env -f infra/compose/docker-compose.local.yml -p webtavern-platform down -v --remove-orphans
docker compose --env-file .env -f infra/compose/docker-compose.local.yml -p webtavern-platform up --build
```

Полный сброс нужен только если требуется заново создать демо-заведения, демо-фото и тестовые бронирования.

## Основные возможности

### Клиентская часть

- регистрация и вход;
- профиль пользователя;
- выбор города проживания;
- автоопределение города через геолокацию браузера;
- каталог заведений с фильтрами;
- режим «рядом со мной»;
- переключение каталога между режимами «Список» и «Карта»;
- карточки заведений с фото-обложками;
- страница заведения с темой оформления;
- галерея заведения со слайдером;
- выбор даты, времени, длительности и количества гостей;
- выбор зала и одного или нескольких столов на схеме;
- отображение занятости столов на выбранный интервал;
- бронирование с hold-резервом выбранных столов;
- отображение необходимости предоплаты;
- профиль с историей броней и оплат;
- отзывы после регистрации, даже если пользователь посещал заведение без бронирования через сайт.

### Кабинет владельца

- просмотр своих заведений;
- создание и редактирование заведения;
- настройка географии, адреса, координат, кухни, ценовой категории и тематики;
- загрузка фотографий заведения;
- выбор фото-обложки;
- конструктор темы заведения;
- готовые цветовые пресеты;
- ручная палитра;
- предпросмотр оформления до сохранения;
- оформление всей страницы настройки по выбранной теме;
- управление залами, столами и схемами;
- настройка правил бронирования и предоплаты;
- управление менеджерами заведения по email;
- ответы на отзывы.

### Менеджерский контур

- доступ только к назначенным заведениям;
- просмотр броней по своим заведениям;
- подтверждение, отмена и обработка броней;
- работа с отзывами и операционными действиями в рамках доступных заведений.

### Уведомления

- центр уведомлений в интерфейсе;
- email-дубликаты через SMTP;
- SMS-дубликаты через SMS.RU;
- журнал доставок;
- кликабельные уведомления с переходом к брони, отзыву, оплате или заведению;
- отметка уведомления прочитанным при открытии.

### Карты

Каталог поддерживает режим карты. Метки строятся по координатам заведений и учитывают текущие фильтры каталога.

Поддерживаемые режимы карт:

- `MAPS_PROVIDER=yandex`;
- `MAPS_PROVIDER=google`;
- `MAPS_PROVIDER=local`.

Если внешний API не загрузился, система показывает локальную карту, чтобы демонстрация проекта не ломалась.

### Платежи

В проекте реализован учебный платёжный сценарий:

- бронь может требовать предоплату;
- после создания брони появляется блок оплаты;
- в профиле отображается срок оплаты;
- успешная учебная оплата переводит бронь дальше по сценарию;
- отмена оплаты отменяет бронь;
- если срок оплаты истёк, бронь отменяется, а слот освобождается.

Боевой эквайринг можно подключить через настройки YooKassa, но для дипломной демонстрации используется безопасный учебный режим без реального списания средств.

## Стоимость бронирования и акции

Владелец может открыть страницу редактирования заведения и настроить необязательный блок «Акции». В нём доступны два типа правил:

- цена по количеству выбранных столов, например 1 стол — 600 RUB, 2 стола — 1100 RUB;
- цена за бронь конкретного зала целиком.

Если правило не настроено, бронирование работает по обычной логике без обязательной стоимости, кроме базовой предоплаты в правилах бронирования.

Клиент на странице заведения может выбрать режим «Выбрать столы» или «Забронировать зал целиком». При брони зала система проверяет занятость всех активных столов этого зала и блокирует весь зал на выбранный интервал.

## Быстрая проверка сценариев

1. Войти владельцем `owner@webtavern.local`.
2. Открыть `/owner/`.
3. Проверить карточки заведений и фото-обложки.
4. Открыть редактирование заведения.
5. Проверить смену темы заведения и ручные цвета.
6. Проверить загрузку фотографии и выбор обложки.
7. Добавить менеджера по email.
8. Выйти и войти менеджером.
9. Проверить, что менеджер видит только назначенные заведения.
10. Открыть `/venues/`.
11. Проверить список заведений и режим карты.
12. Открыть страницу заведения.
13. В блоке «Акции» у владельца настроить цену за 1 стол, 2 стола или бронь целого зала.
14. На клиентской странице выбрать дату, время, зал и один или несколько свободных столов.
15. Проверить отдельный режим «Забронировать зал целиком».
16. Создать бронь.
17. Проверить, что выбранные столы или весь зал становятся недоступными на выбранный интервал.
18. Проверить уведомление в центре уведомлений.
19. Открыть уведомление кликом по карточке.
20. Проверить email/SMS-журнал доставки.
21. Если бронь требует оплату по акции или базовой предоплате, пройти учебный сценарий оплаты.
22. Оставить отзыв после входа в аккаунт.
23. Проверить сортировку отзывов и фильтр «только с фото».
24. Проверить те же основные страницы на мобильной ширине.

## Live-обновление занятости столов

В клиентском сценарии бронирования используется WebSocket `/ws/notifications/`. После создания hold-резерва backend отправляет событие в группу заведения, а все открытые страницы этого заведения сразу заново запрашивают доступность столов для выбранного интервала. Поэтому при бронировании на одном устройстве выбранные столы почти сразу становятся недоступны на других устройствах без ручного обновления страницы.

Realtime-события также отправляются при подтверждении резерва, отмене брони, переносе брони, успешной или отменённой оплате. Nginx настроен на проксирование WebSocket через `Upgrade`/`Connection`.
