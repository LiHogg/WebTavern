# Финальная проверка мобильной адаптации и README

## Проверяемые ширины

- 1440 px — обычный рабочий стол;
- 1024 px — планшет / узкий ноутбук;
- 768 px — планшетная граница;
- 430 px — крупный смартфон;
- 360 px — минимальная мобильная ширина.

## Страницы для проверки

1. `/` — главная страница.
2. `/venues/` — каталог в режиме списка.
3. `/venues/#catalog-map` — каталог в режиме карты.
4. `/venues/<slug>/` — страница заведения.
5. `/venues/<slug>/reviews/` — отзывы.
6. `/account/` — профиль, брони и оплаты.
7. `/notifications/` — уведомления.
8. `/owner/` — кабинет владельца.
9. `/owner/venues/<slug>/edit/` — настройка заведения.
10. Менеджерский кабинет.

## Критерии

- нет горизонтальной прокрутки всей страницы;
- header не разваливается, меню открывается через бургер;
- кнопки не вылезают за экран;
- формы читаемы и не сжимаются до нерабочего состояния;
- карта не ломает ширину страницы;
- слайдер фотографий управляется на мобильной ширине;
- схема зала помещается в экран и остаётся прокручиваемой внутри своего блока;
- уведомления и карточки оплат переносят длинный текст;
- таблицы и списки имеют локальную горизонтальную прокрутку, если места недостаточно;
- README содержит актуальные команды запуска без обязательного `cp .env.example .env`.
